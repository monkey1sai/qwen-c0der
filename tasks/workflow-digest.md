# Workflow Digest

## Six-Step Flow
1. Plan the smallest verifiable change.
2. Implement with focused edits.
3. Debug locally for obvious breakage.
4. Test with the smallest reliable checks.
5. Verify with reproducible evidence.
6. Review and record follow-up work.

## Agent Roles
- Planner: scope and assumptions
- Coder: practical implementation
- Reviewer: risk and defect detection
- Tester: explicit verification evidence

## MCP Strategy
- Keep OpenAI docs available for official product guidance.
- Enable third-party MCP tools only on demand.

## Completion Rule
- Do not mark work complete without reproducible verification.
