# Workspace Context

`.workspace/` stores compact project context for future sessions.

`agents/` stores small role definitions for planning, implementation, review, and testing.

`skills/` is reserved for repo-local reusable skills if this project later needs them.

`repos/` is reserved for linked or nested repositories if the workspace expands.

## Principles
- Keep context small and practical.
- Reuse existing project patterns before adding new instructions.
- Prefer reversible changes with explicit verification.
